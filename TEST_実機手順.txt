【Amazon Price Search 実機テスト手順（Windows 11 + Excel）】

目的:
- 上司PCで「初回セットアップ」「毎回更新」が実際に動くかを最短で確認する。

事前準備:
1) このリポジトリをPC上の任意フォルダに配置（例: デスクトップ配下）
2) Microsoft Excel（デスクトップ版）が起動できることを確認
3) SP-API用の client_id / client_secret / refresh_token と AWS IAM の aws_access_key_id / aws_secret_access_key（必要なら aws_session_token）を手元に用意
4) Excelをすべて閉じる

----------------------------------------
■ テスト1: 初回セットアップ確認（run_init.bat）
----------------------------------------
手順:
1) ルートの run_init.bat をダブルクリック
2) 画面の指示に従い client_id を入力
3) client_secret を入力（表示されません）
4) refresh_token を入力（表示されません）
5) aws_access_key_id を入力
6) aws_secret_access_key を入力（表示されません）
7) aws_session_token を入力（任意、不要なら空欄のままEnter）
8) 「保存完了」メッセージを確認

期待結果:
- secrets\lwa_secrets.xml が作成される
- 例外終了しない

失敗時の確認:
- 入力値の貼り間違いがないか
- セキュリティソフトがPowerShell実行をブロックしていないか

----------------------------------------
■ テスト2: 最小データで更新確認（run_update.bat）
----------------------------------------
入力ファイル作成:
1) data\input.xlsx を作成
2) 1行目に見出し（任意）
3) B2セルに有効なJANを1件入れる
4) B3セルに同じJANを入れる（重複JANキャッシュ確認用）
5) B4セルは空欄にする（空欄処理確認用）
6) 保存してExcelを閉じる

実行:
1) run_update.bat をダブルクリック

期待結果:
- data\output.xlsx が作成される
- C列にASIN、D列に最安価格、E列に取得日時（ISO8601）が入る
- B2/B3が同一JANでも処理全体は完走する
- logs\run.log に開始/完了ログが追記される

失敗時の確認:
- 「run_init.bat を先に実行」→ secrets\lwa_secrets.xml が無い
- 「Excelを閉じてください」→ input/output を開いているExcelを閉じる
- invalid_grant → run_init.batで認証情報を再登録
- 401/403 → SP-API認証設定（権限・方式）を再確認

----------------------------------------
■ テスト3: エラー継続動作の確認
----------------------------------------
手順:
1) data\input.xlsx のB列に無効JANを1件混ぜる
2) run_update.bat を実行

期待結果:
- 無効JAN行は C/D が空欄、Eは時刻が入る
- ほかの有効JAN行は処理される（全体停止しない）
- logs\run.log に該当行のエラーログが追記される

----------------------------------------
■ 合格判定（最低限リリース可）
----------------------------------------
次の4点を満たせば「今週配布版として最低限OK」:
[ ] run_init.bat が完了し secrets\lwa_secrets.xml が生成される
[ ] run_update.bat で output.xlsx が生成される
[ ] エラー行があっても全体処理が継続する
[ ] 主要トラブル（Excel開きっぱなし / invalid_grant / 401/403）の対処が運用者で可能
